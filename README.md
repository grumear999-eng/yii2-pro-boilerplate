## 📖 О проекте (About)

Это внутренний корпоративный стартовый шаблон (Boilerplate) нашей команды, спроектированный на базе **Yii2 Advanced Template**. Мы используем этот репозиторий как фундамент для быстрого старта новых коммерческих проектов (ERP, CRM систем, личных кабинетов).

Здесь соблюдены все наши внутренние стандарты написания кода:
- Строгое следование **PSR-12**.
- Паттерн **"Толстые модели — тонкие контроллеры"** (вынесение логики фильтрации в `ActiveQuery` классы).
- Использование возможностей фреймворка на 100%: `Behaviors`, `Events`, `Dependency Injection`.
- Подготовленное окружение для разработки REST API для взаимодействия с современными Frontend-приложениями (React, Vue) через микросервисную архитектуру.

## 🏛 Архитектурные особенности сборки

### 1. `common/models` (Доменная логика)
Мы не храним `where()` запросы в контроллерах. Все запросы инкапсулированы в кастомные `ActiveQuery` классы (например, `OrderQuery.php`). Это позволяет переиспользовать сложную логику выборок и сохранять читаемость кода:
```php
// Пример использования в приложении
$pendingOrders = Order::find()->byUser($userId)->unpaid()->active()->all();
```

Мы активно используем `Behaviors` (`TimestampBehavior`, `BlameableBehavior`) для автоматического ведения лог-полей. Разумеется, в боевых проектах добавляется паттерн Service Layer для выноса бизнес-логики (например `OrderPaymentService`), когда сущности становятся слишком "тяжелыми".

### 2. `api/controllers` (API First)
В сборку внедрены настроенные `ActiveController` (классический REST API Yii2). 
Уже настроены:
- **CORS-фильтры**, чтобы избежать блокировок OPTIONS-запросов браузером при работе с React/SPA.
- **HttpBearerAuth** для секьюрной JWT/Token авторизации.
- Переопределен `prepareDataProvider`, чтобы данные через REST автоматически фильтровались согласно правам доступа (RBAC).

## 🚀 Стек и Инфраструктура
- **Фреймворк:** Yii Framework 2 (v2.0.48+)
- **PHP:** 8.1+
- **База данных:** PostgreSQL / MySQL абстракция
- **Кеширование / Очереди:** Redis (`yii2-queue`, `yii2-redis`)
- **Frontend-интеграция:** JSON REST API / Guzzle

## 🔐 Комментарий разработчика (Почему так?)
Код в этом репозитории не раздут лишними плагинами. Это чистый, строгий и поддерживаемый скелет приложения, который легко масштабируется под любые задачи коммерческого энтерпрайза. Подобный подход позволяет нам гарантировать высокую скорость разработки на стороне бэкенда без превращения проекта в "legacy" через полгода.
